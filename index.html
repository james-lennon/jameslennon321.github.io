Driftlight

Driftlight is an iOS action/puzzle game that I developed using libgdx, a cross-platform OpenGL framework. The objective is to guide your light through the dark maze and escape without touching the bottom. Right now there are 40 levels of increasing difficulty to complete.

  • Date: August 2014
  • Type: iOS App

Nebulous

Nebulous is a multiplayer OpenGL game I created for iOS (you can downlod it on the App Store here). There are three gameplay modes: single player, online multiplayer, and bluetooth multiplayer. In singleplayer mode, I developed artificially intelligent enemy ships for the player to fight. I used Apple's Game Center for online multiplayer, and Multipeer Connectivity for bluetooth multiplayer.

  • Date: May 2014
  • Type: iOS App

HW Assassin

At Harvard-Westlake school, the students run a school-wide game of assassin (a game where each player is given another player that they have to sneak up on and "kill"). To add useful features and reduce the work necessary to run the game, I created an iOS & Android app, as well as a web panel for the game administrators.

  • Type: iOS, Android, & Web App
  • Date: June 2014

Algebra Engine

The Algebra Engine is a Java Library I created to help me with my math homework. I developed a parsing algorithm (based on the Shunting-yard algorithm) to read and interpret math expressions from the input, and it can them simplify and apply operations to the math expression (or take the derivative). It is particularly helpful with simplifying or finding the derivative of complicated polynomial or rational expressions.

  • Type: Java Library
  • Date: December 2013

Terrain Generator

After learning about biomes in AP Biology, I wanted to see if I could make an algorithm to randomly generate realistic-looking terrain. After researching and optimizing the variables, I developed a sufficiently realistic terrain generator. It uses coherent noise generation to generate moisture and elevation maps, and then combines them to create the appropriate biomes and fill them with randomized detail. I then made a simple game that uses the algorithm to generate random worlds in which the player can walk around and fight enemies.

  • Type: Algorithm & Java Game
  • Date: July 2013

Smart Controller

When I did an internship at UCLA's Smart Grid Energy Research Center (SMERC), I developed an Android app under the supervision of a grad student. The app monitors and controls the electrical usage of appliances in their lab. It was part of an experiment for future "smart homes" that can automatically adjust their power usages to maximize efficiency.

  • Type: Android
  • Date: June 2012

Openpad

Openpad is a C++ library I developed to easily allow games to use smartphones as remote controllers. I led the project and created the C++ server and game library, and my partner developed an iOS remote control app. It uses a JSON-based messaging protocol to quickly communicate between the game and the controllers. One of the coolest features is that the controls are completely dynamic and the controller layout is designied and updated programmatically. Using this library, I created a multiplayer pong game, a racing game (with an algorithm I created to programmatically create racetracks), and I used it as a remote control for a drone.

  • Type: C++ Library & iOS App
  • Date: November 2014

Campus Checkout

`

As a member of my high school's Tech Council, I developed a new attendance checkout system. The project was to modernize the school's checkout system so that instead of writing their names on a clipboard, students could simply scan their ID cards to leave campus.
